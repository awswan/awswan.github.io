Mastodon tips

This is a collection of points about posting on Mastodon, that I didn't find obvious and aren't always listed in online guides. Most are from the technical documentation, but there are a few I found through trial and error, and am not 100% certain about. I focus on mathstodon, but most points apply to any server. Similarly, I focus on the web interface because that's what I use.

Posting

  1. Markdown is not currently supported.
  2. URLs are automatically made into links as long as they start http://.
  3. On mathstodon use \(...\) to write a formula inline and \[...\] for display math. Formulas are rendered using MathJax, the same system used on mathoverflow and many other sites.
  4. You can convert formulas to unicode from the edit box. If you want to do this for a formula that already has text after it (if you've copied and pasted from elsewhere, for example) you can do this by making a trivial edit to the formula to activate the unicode conversion button.
  5. The conversion to unicode is not always possible. This seems to include anything with subscripts.
  6. Formula rendering is not available on all mastodon clients. Unicode is more widely supported.
  7. MathJax is potentially better for screen readers than unicode.
  8. The CW button is for content warnings. This splits a post into two parts. The second part is hidden by default and can be displayed by clicking "Show more." Content warnings are used for a range of things from sensitive or potentially offensive content through to spoilers for film endings or maths problems. It's common on mastodon to use content warnings for opinions on any divisive topic such as politics or religion.
  9. If you reply to a post with a content warning, the same content warning applies to your reply by default. Disable it by clicking CW if you don't want this.
  10. If you attach a picture, you can use the editing function to add alt text describing it. You should do this. It is easy and helps anyone who can't see the image easily (visually impaired users, slow internet connections, etc).

Character count

  1. URLs always count as 23 characters. When the post is displayed the URL is shortened to a reasonable length.
  2. In mentions only the username counts towards the character count, not the server.
  3. In content warnings the character counts of the warning and content are added together - the total must be below the limit of 500 characters.
  4. Formulas take up less characters if you convert them to unicode.
  5. Paragraph breaks only take up two characters but have a big visual effect (in my opinion) in making the post seem longer. This can help readability, but also takes up more space in timelines.

Search and indexing

  1. Hashtags are not case sensitive in search. E.g. searching for #HoTT will also bring up posts containing #hott. Capitalising the start of each word helps users with screen readers and makes the hashtag more readable for everyone, but has no effect on indexing.
  2. Full text search is available on mathstodon, but will only return posts you authored, favourited, bookmarked or are mentioned in. This is a deliberate design decision by the mastodon developers.

Publishing level

  1. Four publishing levels are available: public, unlisted, followers only, and mentioned only. Level can be selecting from the editing box.
  2. On the web interface the publishing level of a post is displayed as an icon in the upper right corner of the post.
  3. Unlisted posts are publicly accessible, but do not appear on timelines. It's common to use this for long threads and followup discussion.
  4. Mentioned only posts are also called direct messages. They appear both in the home timeline and the direct message timeline. They are not end-to-end encrypted.

Favourites/bookmarks/boosting

  1. Favourites are "public bookmarks." They are visible to everyone.
  2. When you favourite a post, the author is sent a notification.
  3. You can see the full list of people who have favourited a post on the web interface by clicking on the post to select it, and then clicking the faint "star icon" below the text, but above the other star icon in the toolbar.
  4. Bookmarks are private but otherwise work the same as favourites.
  5. Unlike twitter, favourites are not used in any kind of recommendation algorithm. If you want a post to be more prominent, boost instead.
  6. Boosts will relay the post to your followers if they haven't seen it already. If you have followers on another server, or followers that haven't checked the local timeline recently, this might be the only way they hear about the post.

Threads

  1. Internally, each post either starts a new thread or has a link to a parent post that it replies to. Hence posts are structured as a forest, with a tree for each thread.
  2. The tree structure is not visually indicated in the user interface.
  3. On the website, clicking on a post to select it displays the post itself, ancestors and descendants. It does not display "siblings" or "cousins."
  4. Because of the above, if you want clicking on a post to show the entire thread you will need to make the tree into a linear order by making each post a reply to the previous one.

Profile page

  1. URLs listed in the profile metadata can be displayed in green with a tick to indicate they are verified. To do this, add a link back to your profile page, with the attribute rel="me". Example HTML is provided on the profile settings page.
  2. The timeline on your profile page displays posts you have posted public and posts you have boosted. It does not display posts you authored but made unlisted.
  3. It is possible to "pin" posts you have authored to the top of your profile page, by seleting "Pin on profile" from the 3 dots menu at the bottom of a post.
